  ### Collection: Non-Code & Design Work

Rolling (zero-downtime) updates in Coolify require a single image built from a Dockerfile, a passing health-check endpoint, and the default container naming. They **break** with multi-service Docker Compose deployments and with host port mappings (which disable the health check). So: one deployable app = one container with a real `/health` route, default naming, no port mapping. Keep Postgres, Redis and Unleash as *separate* Coolify resources — which also stops an app deploy needlessly restarting your database.
